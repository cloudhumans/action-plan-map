Objetivo

Este dashboard mapeia todos os projetos ativos da Cloud Humans e serve como ponto central para priorizar onde executar planos de acao (skill /ch-action-plan). Cada card e um projeto que pode ter seu plano de acao executado, gerando um diagnostico com acoes priorizadas e estimativas de impacto.

Categorias de saude

CategoriaRetencao N1Significado
Critico< 30%Projeto em risco. Menos de 1/3 dos tickets resolvidos pela ClaudIA. Requer acao imediata.
Atencao30% a 40%Abaixo do esperado. Ha espaco significativo para melhoria com acoes focadas.
Razoavel40% a 50%Funcionando mas com potencial. Otimizacoes podem gerar ganhos relevantes.
Bom50% a 70%Performance saudavel. Ajustes finos para maximizar valor.
Excelente70%+Performance de referencia. Manter e buscar otimizacoes incrementais.

Regua de conversao por tipo de acao

Quando um plano de acao e executado, cada acao recebe uma estimativa de impacto baseada no numero de tickets afetados e uma taxa de conversao que varia por tipo:
Tipo de acaoTaxa de conversaoJustificativa
Erro tecnico (Eddie, timeout, bug)60%Problemas tecnicos sao geralmente corrigiveis de forma direta
ProblemResolvedChecker / FUP50%Metade dos FUP negativos podem ser retidos com ajuste de fluxo
Clarificacao / classificacao35%Depende de qualidade da IDS e linguagem do usuario
Novo conteudo (no_relevant_content)60%Conteudo inexistente: ao criar, alto % de captura
Conversao de conteudo N2 para N115%Conservador — muitos N2s existem por razao valida
Loop / secao repetida40%Conteudo parcial que pode ser completado
Projetos sem plano executado nao possuem estimativa de impacto

Marcadores na barra de retencao

Cada card possui uma barra de retencao com ate 3 elementos: a retencao atual (barra colorida), a meta do Airtable (marcador roxo), e o potencial com acoes (marcador amarelo, exibido apenas para projetos com plano executado = retencao atual + impacto estimado das acoes).

Tierizacao CS

Cada projeto possui um Tier vindo do Airtable (Accounts → Tierizacao CS). Tier 1 = contas estrategicas de maior valor. Tier 2 = contas intermediarias. Tier 3 = contas menores ou em early stage. Use o filtro de Tier para priorizar analises.

Fontes de dados

Os dados vem de: Metabase (volume, retencao, N2s dos ultimos 30 dias), Airtable CRM (meta de retencao, CS Owner, status, Tier da conta), e sessoes anteriores (planos de acao ja executados com a skill /ch-action-plan).

Como usar

1. Filtre por CS Owner para ver seus clientes. 2. Ordene por retencao (menor) ou impacto (maior) para priorizar. 3. Para qualquer card pendente, rode /ch-action-plan [nome do projeto] para gerar o plano completo. 4. Os resultados serao adicionados automaticamente ao dashboard.
